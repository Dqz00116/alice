# Multi-Turn CLI and Tool Support for Anthropic Provider

## Feature 1: Pass Tool Definitions to the Provider

- Update `StreamingProvider::format_messages` trait signature to accept `tools: &[ToolDef]`.
- Update `AnthropicProvider::format_messages` to serialize tools as:
  ```json
  {
    "name": "...",
    "description": "...",
    "input_schema": { ... }
  }
  ```
  and include them in the request body under the `tools` key.
- Update `EffectExecutor::CallLLM` to read `ToolsComponent.definitions` and pass them to `format_messages`.
- Update all mock/test providers to match the new trait signature.

## Feature 2: Multi-Turn CLI Loop

- Wrap the existing event-processing loop in an outer loop inside `main.rs`.
- After the inner event loop finishes one assistant turn, prompt for the next user input.
- If the user types `/exit` or `/quit`, stop the outer loop.
- If the user types an empty line, continue prompting (skip producing an event).
- Respect `LoopComponent.should_continue` and `abort_manager.is_aborted()`.
- Print a friendly goodbye message when exiting.

## Feature 3: Tests

- Add a unit test in `crates/alice-providers/tests/anthropic_test.rs` that verifies `format_messages` includes `tools` when definitions are provided.
- Add a CLI integration test in `crates/alice/tests/cli_loop_test.rs` that simulates two user inputs and verifies two assistant replies are appended to `MessagesComponent`.
- Update existing tests affected by the trait signature change.
