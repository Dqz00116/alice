# End-to-End Anthropic Tool Calling and Core Toolset

## Feature 1: Anthropic `tool_use` SSE Parsing

- Extend `AnthropicProvider::stream_chat` to handle:
  - `content_block_start` with `type: "tool_use"` → start tracking a new tool call.
  - `content_block_delta` with `partial_json` → accumulate JSON fragments.
  - `content_block_stop` → finalize the tool call and emit `LLMStreamEvent::ToolCall`.
- Text and thinking deltas continue to work as before.

## Feature 2: Anthropic-Compatible Message Serialization

- `Message::Assistant` with tool calls serializes to `role: "assistant"`, `content` array containing a text block plus `tool_use` blocks.
- `Message::Assistant` without tool calls keeps the existing string `content`.
- `Message::Tool` serializes to `role: "user"`, `content` array containing a single `tool_result` block with `tool_use_id` and `content`.

## Feature 3: Core Toolset

Add the following tools in `alice-tools`, each with a JSON schema and a handler:

- `Bash` — execute a shell command via `bash -c` inside the project root, with optional timeout and output truncation.
- `FileRead` — read a text file, optionally paginated by `offset` and `limit`.
- `FileWrite` — write content to a file, creating parent directories.
- `FileEdit` — replace `old_string` with `new_string` in a file.
- `Glob` — list files matching a glob pattern under a directory.

All file tools resolve paths relative to the project root and reject paths that escape it.

## Feature 4: CLI Registration

- Register the new tools alongside `echo` in `crates/alice/src/main.rs`.

## Feature 5: Tests

- SSE parsing tests for tool-use events.
- Serialization tests for assistant tool-use and tool-result messages.
- Unit tests for each new tool handler.
- Update existing tests affected by message serialization changes.
