## Description

将 Alice 从 TypeScript 完整重写为 Rust，保留其核心设计哲学（数据驱动、无状态、流式并发、高度可定制），同时利用 Rust 的所有权系统、类型系统和零成本抽象，重新设计接口以消除 TS 版本中的权衡取舍（如 snapshot 共享引用约定）。

Alice 是一个 CLI AI Agent 引擎，用于驱动 LLM 交互并编排工具调用。目标用户是需要构建自定义 AI Agent 的开发者，他们通过组装 World、注册 System、配置 Provider 和安装 Middleware 来构建自己的 Agent 应用。

## Acceptance Criteria

- [ ] `alice-core` crate 提供完整的引擎骨架：World、System trait、Event 类型、Effect 类型、EventBus、EffectExecutor、SystemRegistry
- [ ] `alice-providers` crate 提供至少一个可用的 LLM Provider 实现（Anthropic），`StreamingProvider` trait 可被第三方实现
- [ ] `alice-tools` crate 提供 ToolDef/ToolHandler 抽象和至少一个内置工具
- [ ] `alice` CLI binary 装配所有组件，启动后可接收用户输入、驱动 LLM 流式输出、执行工具调用、渲染结果到终端
- [ ] 核心闭环路径完整：用户输入 → LLM 流式输出 → 工具调用 → 结果返回 → LLM 继续 → 终止
- [ ] 所有公开 API 使用 Rust trait + generics 实现编译期类型安全，无 `Box<dyn Any>` 或运行时 downcast
- [ ] 异步基于 tokio 运行时，LLM 流式输出使用 `AsyncGenerator` 等价物（`Stream` / `async fn` 配合 channel）
- [ ] 错误处理使用 thiserror（库层）+ anyhow（CLI 层）
- [ ] `cargo test` 全部通过，`cargo clippy` 零警告
- [ ] 工作空间结构：`alice-core` (lib)、`alice-providers` (lib)、`alice-tools` (lib)、`alice` (bin)

## Notes

- 保留原 TS 版的五层架构思想（Host → Core → Business → Data → Infrastructure），但用 Rust 惯用方式重新表达
- World 用 trait + generics 替代动态组件存储：`trait World { type Components; fn get<C>(&self) -> &C; }`
- System 保持纯函数签名：`Fn(&Snapshot, &Event) -> Vec<Effect>`
- Effect 保持声明式描述风格，EffectExecutor 负责执行
- 不引入 bevy_ecs 等重型依赖，保持最小依赖树
