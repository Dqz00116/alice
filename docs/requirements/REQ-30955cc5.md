# End-to-End Anthropic Tool Calling and Core Toolset

## Problem

Although the engine has a tool execution pipeline and tool definitions are now sent to Anthropic, the real end-to-end flow is still broken:

1. `AnthropicProvider` cannot parse `tool_use` SSE events, so model-initiated tool calls never reach `ToolSystem`.
2. `Message::Assistant` and `Message::Tool` are serialized using an OpenAI-style shape (`tool_calls` array and `role: "tool"`) instead of Anthropic's expected content-block shape (`tool_use` / `tool_result` blocks inside `content`).
3. The only available tool is `echo`, which is not useful for real agent work.

## Goals

- Parse Anthropic `tool_use` SSE events and emit `LLMStreamEvent::ToolCall`.
- Serialize assistant tool-use messages and tool-result messages in Anthropic format.
- Add a core set of file/system tools inspired by the Claude Code tool schemas: `Bash`, `FileRead`, `FileWrite`, `FileEdit`, `Glob`.
- Keep all tools sandboxed to the project working directory.
- Register the new tools in the CLI.
- Add unit tests for SSE parsing, message serialization, and each new tool.

## Non-Goals

- Reimplement every Claude Code tool (web search, MCP, subagents, notebooks, etc.).
- Cross-provider compatibility beyond Anthropic.
- Advanced sandboxing (network restrictions, filesystem allow-lists).
- Background task management.

## Acceptance Criteria

- [ ] Anthropic SSE parser emits `LLMStreamEvent::ToolCall` for `content_block_start/stop/delta` with `type: "tool_use"`.
- [ ] `Message::Assistant` with tool calls serializes to Anthropic content blocks (`tool_use` + text blocks).
- [ ] `Message::Tool` serializes to a `user` message with a `tool_result` content block.
- [ ] New tools `Bash`, `FileRead`, `FileWrite`, `FileEdit`, `Glob` are implemented and registered in the CLI.
- [ ] New tools reject paths outside the project working directory.
- [ ] New tests cover tool-use SSE parsing, Anthropic message serialization, and each new tool.
- [ ] `cargo test --all-targets`, `cargo clippy --all-targets`, and `cargo build --release` all pass.
